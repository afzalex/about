About fzsystems

fzsystems is a service platform under which various fz* services operate, including fzbox, fzmacair, fzryzen, and other fz* services. fzsystems provides centralized authentication and authorization for all services within its ecosystem.

fzsystems is registered as a Google Cloud Console project and uses Google OAuth for authentication across all fz* services. Each device in the fzsystems network (such as the Raspberry Pi server running fzbox, the laptop running fzmacair, and the desktop running fzryzen) runs a Traefik reverse proxy server configured with OpenID Connect (OIDC) authentication that integrates with fzsystems.

When users attempt to access any fz* service, Traefik redirects them to Google OAuth through the fzsystems Google Cloud Console project. Once authenticated, users gain access to the requested service, and the same authentication session can be used across all fz* services (Single Sign-On). This centralized approach ensures consistent security policies and user management across the entire fzsystems infrastructure.

What is fzsystems?

fzsystems is a service platform that hosts and manages various fz* services. It is registered as a Google Cloud Console project and serves as the centralized authentication provider for all fz* services operating under it.

Services under fzsystems include fzbox, fzmacair, fzryzen, and other fz* services, each running on different devices in the infrastructure. Each device runs a Traefik reverse proxy server that handles routing, SSL/TLS termination, and authentication. The Traefik servers are configured with OIDC (OpenID Connect) authentication, which connects to the fzsystems Google Cloud Console project for user authentication.

This architecture allows fzsystems to provide a unified authentication experience across all fz* services. Users authenticate once through Google OAuth via fzsystems, and that authentication is recognized across all services in the network, eliminating the need for separate login credentials for each service.

All fz* services are registered as authorized applications within the fzsystems Google Cloud Console project, ensuring they can securely authenticate users through the centralized system. This setup provides a scalable and secure way to manage access to multiple services across different devices while maintaining consistent security policies.

Security and Privacy

fzsystems is designed with security and privacy in mind:

  • All authentication is handled through Google's secure OAuth 2.0 / OIDC infrastructure
  • No passwords are stored locally - all authentication is delegated to Google
  • OIDC tokens are used for session management, providing secure, time-limited access
  • All communications are encrypted using HTTPS/TLS
  • Access can be revoked at any time through Google account settings

For more information about how we handle your data, please see our Privacy Policy.

Authentication Technology

fzsystems uses Google OAuth 2.0 and OpenID Connect (OIDC) for authentication. The service is registered as a Google Cloud Console project, enabling secure Single Sign-On (SSO) across all fz* services. Each device hosting fz* services runs a Traefik reverse proxy server that handles authentication, routing, and SSL/TLS termination.

All fz* services are registered as authorized applications within the fzsystems Google Cloud project, ensuring consistent and secure authentication across all services.
